# Руководство по обслуживанию SSL-сертификатов для edfuncontent.com

## Подготовка

Загрузите все скрипты на сервер:

```bash
scp renew-ssl.sh setup-auto-renewal.sh check-portal-status.sh fix-login-502.sh root@198.177.123.158:/root/ssl-scripts/
```

Сделайте их исполняемыми:

```bash
ssh root@198.177.123.158 "chmod +x /root/ssl-scripts/*.sh"
```

## Обновление SSL-сертификата вручную

Если возникла необходимость обновить сертификат вручную, выполните:

```bash
ssh root@198.177.123.158 "/root/ssl-scripts/renew-ssl.sh"
```

## Настройка автоматического обновления

Для настройки автоматического обновления сертификата выполните:

```bash
ssh root@198.177.123.158 "/root/ssl-scripts/setup-auto-renewal.sh"
```

Это добавит задание в crontab для ежедневного обновления сертификата в 3:00.

## Проверка состояния портала

Для проверки состояния всех компонентов портала выполните:

```bash
ssh root@198.177.123.158 "/root/ssl-scripts/check-portal-status.sh"
```

## Исправление ошибки 502 при логине

Если возникает ошибка 502 Bad Gateway при попытке входа в систему, выполните:

```bash
ssh root@198.177.123.158 "/root/ssl-scripts/fix-login-502.sh"
```

## Решение распространенных проблем

### 1. Истёк срок действия сертификата

**Признаки:**
- Предупреждение о небезопасном соединении в браузере
- Ошибки 502 Bad Gateway при запросах к API

**Решение:**
```bash
ssh root@198.177.123.158 "/root/ssl-scripts/renew-ssl.sh"
```

### 2. Проблемы с доступом к API

**Признаки:**
- Фронтенд загружается, но функции, требующие API, не работают
- Ошибки 502 Bad Gateway при запросах к API

**Решение:**
```bash
ssh root@198.177.123.158 "systemctl restart php8.1-fpm && systemctl restart nginx"
```

### 3. Полная недоступность портала

**Признаки:**
- Сайт не открывается вообще

**Решение:**
1. Проверьте состояние:
   ```bash
   ssh root@198.177.123.158 "/root/ssl-scripts/check-portal-status.sh"
   ```
2. Запустите скрипт исправления:
   ```bash
   ssh root@198.177.123.158 "/root/ssl-scripts/fix-login-502.sh"
   ```

## Важно помнить

1. SSL-сертификаты Let's Encrypt действительны 90 дней
2. Автоматическое обновление должно происходить за 30 дней до истечения срока
3. При изменении домена потребуется новый сертификат
4. Для мониторинга рекомендуется добавить оповещения о состоянии сертификата

## Проверка конфигурации HTTPS

Для обеспечения безопасной работы необходимо проверить, что фронтенд использует HTTPS для всех API-запросов:

```bash
ssh root@198.177.123.158 "grep -r 'http://' /var/www/portal/frontend --include='*.js'"
```

Если найдены HTTP-запросы, их необходимо заменить на HTTPS.
